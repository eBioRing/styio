# Syntax Design

**Purpose:** Define the compact active-syntax entrypoints under `docs/design/syntax/`; generated inventory lives in [INDEX.md](./INDEX.md), while full grammar, tokens, and semantics stay in owner documents.

**Last updated:** 2026-05-09

## Scope

1. Store compact active-syntax maps and narrow syntax reference pages here.
2. Keep full grammar in [../Styio-EBNF.md](../Styio-EBNF.md).
3. Keep token names and disambiguation in [../Styio-Symbol-Reference.md](../Styio-Symbol-Reference.md).
4. Keep full semantic discussion in [../Styio-Language-Design.md](../Styio-Language-Design.md).
5. Keep resource topology and safety rules in [../Styio-Resource-Topology.md](../Styio-Resource-Topology.md).
6. Do not add retired-syntax catalogs here; use Git history for exact old wording.

## Naming Rules

1. Use stable, searchable uppercase filenames for compact syntax tables.
2. Start broad navigation in [ACTIVE-SYNTAX.md](./ACTIVE-SYNTAX.md), then link to the owning SSOT.
3. Do not add broad design narratives here.
4. Put compatibility aliases after canonical forms, not before them.
