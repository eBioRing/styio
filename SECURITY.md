# Security Policy

Report security-sensitive Styio issues privately when possible. If the hosting
surface does not expose private vulnerability reporting, open a minimal issue
requesting a private contact path and do not include exploit details.

Include:

- affected commit or branch
- affected command, source file, or input shape
- expected and observed behavior
- sanitizer, fuzz, or crash artifact when available

Security fixes should include a regression test or a documented reason why a
test cannot be added in the same change.
